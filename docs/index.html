Problem

Why this benchmark exists

Surgical AI systems need reliable detection and tracking of instruments across challenging video conditions. This project benchmarks detector and tracker performance on CholecTrack20 to study how well models handle surgical tool localization and multi-object tracking.

01

Detection quality

Measure whether models can locate surgical tools accurately across tools, scenes, and visual difficulty.

02

Tracking continuity

Check whether the tracker maintains identities across visibility, intracorporeal, and intraoperative perspectives.

03

Real project constraints

Report results honestly within the compute, time, and parameter-sweep budget that was available.

Dataset and scope

Benchmark reference, not dataset contribution

Dataset

CholecTrack20

The project uses CholecTrack20 as the benchmark dataset for detector and tracker comparison.

  • This repo is a benchmark reference, not a dataset contribution.
  • Datasets, checkpoints, generated logs, and large exports are intentionally excluded from Git.
  • Small benchmark summaries, configs, and source code are tracked for public review.
Interpretation

How to read the results

The numbers on this page should be read as real-world project results under compute, time, and parameter-sweep limits, not as an unconstrained global optimum.

Reference context:

The official CAMMA CholecTrack20 repository publishes its own benchmark leaderboards. That context is useful, but this page is a local benchmark story and should not be treated as a direct leaderboard submission.

For reference, the official repo reports much stronger numbers on its own protocol, including detector rows such as YOLOv10 around 80.1 AP0.5 and tracking rows such as Bot-SORT around 69.6 MOTA on the visibility perspective. Those results are useful context, but they are not directly comparable to this local benchmark.

Methodology

End-to-end pipeline

1

Prepare CholecTrack20 data and annotations

Set up the dataset locally and convert the annotation structure needed for the benchmark pipeline.

2

Train and evaluate detector baselines

Run YOLO11 and EfficientDet-D0 experiments under controlled training settings.

3

Compare YOLO11 and EfficientDet-D0 metrics

Review AP, per-class, challenge, and runtime results to identify the stronger detector candidate.

4

Use stronger detector output for tracking

Feed the selected detector into EA-StrongSORT for frame-by-frame surgical tool tracking.

5

Evaluate across three tracking perspectives

Measure visibility, intracorporeal, and intraoperative track continuity under class-aware MOT evaluation.

6

Record compact benchmark summaries

Store the final summaries in `benchmarks/` for review, comparison, and public sharing.

Reproducibility

Setup and smoke checks

The dataset, checkpoints, generated reports, and large exports are not committed. Place the CholecTrack20 dataset and annotations locally before reproducing runs.

Environment 
conda env create -f environment.yml
conda activate deep_learning_project
python -m src.train check-env
Smoke checks 
python -c "import torch; print(torch.__version__, torch.version.cuda, torch.cuda.is_available())"
python -m pip check
python -c "import cv2, boxmot, pycocotools, motmetrics; print('tracking deps ok')"
